# Part 2
## コンテンツ駆動を支える技術の3本柱

---

## 1) Content Layer / Live Collections
- コンテンツの扱いが「ファイル」→「層（Layer）」へ
- Live Content Collectionsで、実行時もASTレベルで型安全な参照ができます。
- 更新・検証・統合まで含めてコンテンツ運用をAstro側で引き受ける

---

## 2) Islands → Server Islands

- 動的UIは必要な島だけを立てればいい
- server:defer（Server Islands）がSSRレスポンスから部分的に遅延レンダリングを許容
- 速さを崩さずに動的要件を足すための現実解が日本の案件にも刺さる
- Astro = コンテンツ駆動サイトのための Server-First フレームワーク
- 公式も「Server-first / Content-driven / Island-driven」を併記
- SSG vs SPAではなく、コンテンツとインタラクションの配分設計


---

## 3) Full-stack DX

- Astroは“薄い”まま、必要な分だけアプリ的振る舞いを足せる
- Actions（型安全なバックエンド関数）、astro:env（厳格な環境変数）、Sessions API（5.7+）
- BFF・フォーム・認証を「MPAの良さ」を保ちながら実装できる
